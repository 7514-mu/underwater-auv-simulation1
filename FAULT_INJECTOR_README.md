# 💉 水下机器人故障注入系统

使用**正确的ApplyLinkWrench格式**进行真实的物理故障注入。

## 🎯 功能特性

### 推进器故障
- **单推进器失效** - 将特定推进器效率设为0%
- **推进器效率降低** - 降低推进器效率至指定值
- **多推进器故障** - 多个推进器同时失效

### 物理环境故障（直接施力）
- **湍流干扰** - 直接对机器人本体施加随机扰动力（高斯分布）
- **磁场干扰** - 直接对机器人本体施加小幅随机扰动力
- **压力舱进水** - 直接对机器人本体施加持续向下的力
- **传感器漂移** - 直接对机器人本体施加恒定漂移力
- **碰撞模拟** - 6个方向的碰撞（通过ApplyLinkWrench）
- **洋流干扰** - 使用UUV的洋流服务

### 传感器故障（修改传感器数据）
- **IMU噪声干扰** - 修改IMU加速度和角速度数据，添加噪声和偏置
- **磁力计干扰** - 修改磁力计磁场数据，添加噪声和偏置
- **水温变化** - 通过修改所有推进器效率模拟密度变化

## 🔑 物理故障实现原理

### ApplyLinkWrench直接施力

**湍流、磁场、压力舱、传感器漂移**故障现在使用`ApplyLinkWrench`服务直接对机器人本体施加物理力：

```bash
ros2 service call /apply_link_wrench gazebo_msgs/srv/ApplyLinkWrench \
"{link_name: 'rexrov/base_link', reference_frame: 'world', \
wrench: {force: {x: 500.0, y: 0.0, z: 0.0}, torque: {x: 0.0, y: 0.0, z: 0.0}}, \
duration: {sec: -1, nanosec: 0}}"
```

**关键要点**：
- 服务路径：`/apply_link_wrench`
- 参数结构：`wrench: {force: {...}, torque: {...}}`（嵌套格式）
- reference_frame：`'world'`（世界坐标系）
- duration：`-1`表示持续施加，或指定秒数

### 传感器噪声注入（IMU和磁力计）

**IMU和磁力计故障**通过订阅原始传感器数据，添加噪声和偏置后重新发布：

**IMU故障**：
- 订阅：`/rexrov/imu/data_raw`（原始数据）
- 发布：`/rexrov/imu/data`（带噪声数据）
- 注入内容：
  - 线加速度噪声（高斯分布）
  - 线加速度偏置（恒定值）
  - 角速度噪声（高斯分布）

**磁力计故障**：
- 订阅：`/rexrov/magnetometer_raw`（原始数据）
- 发布：`/rexrov/magnetometer`（带噪声数据）
- 注入内容：
  - 磁场噪声（高斯分布）
  - 磁场偏置（恒定值）

### 不同故障的力学特征

| 故障类型 | 力的特征 | 强度范围 |
|---------|---------|---------|
| 湍流 | 随机高斯分布力 | 300-1500N |
| 磁场 | 小幅随机力 | 50-200N |
| 压力舱 | 持续向下恒力 | 490-4905N |
| 传感器漂移 | 恒定漂移力 | 100N |

### 传感器故障的数据特征

| 故障类型 | 数据修改 | 强度范围 | 效果 |
|---------|---------|---------|------|
| IMU噪声 | 加速度+角速度噪声 | 加速度±0.01-0.15 m/s²，角速度±0.005-0.1 rad/s | 姿态抖动，路径不平滑 |
| 磁力计干扰 | 磁场噪声+偏置 | 噪声±0.1-0.5 μT，偏置0.5-2.0 μT | 航向偏差，走直线会偏航 |

## ✅ 优势

- ✅ 物理故障使用**真实物理力**（非推进器模拟）
- ✅ 传感器故障修改**真实传感器数据**（加速度、角速度、磁场）
- ✅ 支持**故障组合**（推进器故障 + 环境故障 + 传感器故障）
- ✅ 可自定义强度和持续时间
- ✅ 动态启停故障
- ✅ 基于正确的ApplyLinkWrench格式
- ✅ 传感器故障可在导航中实时测试

## 🔧 技术细节

- 使用 `gazebo_msgs/srv/ApplyLinkWrench` 服务直接施力
- 使用 `sensor_msgs/Imu` 消息类型处理IMU数据
- 使用 `geometry_msgs/Vector3Stamped` 消息类型处理磁力计数据
- 多线程支持动态故障（湍流、磁场等持续运行）
- 推进器故障通过 `SetThrusterEfficiency` 服务实现
- 传感器故障通过订阅-发布模式实现（raw data → noisy data）
- 环境故障、推进器故障、传感器故障可独立控制和叠加
