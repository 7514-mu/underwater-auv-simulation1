# 🧭 惯性导航系统（INS）实现指南

## 📚 什么是惯性导航？

**惯性导航（Inertial Navigation）** 是利用惯性传感器（加速度计和陀螺仪）来测量运动体的加速度和角速度，再计算出其位置、速度和姿态等运动状态的技术。

### **核心原理**

```
加速度计 → 测量加速度 → 积分 → 速度 → 积分 → 位置
陀螺仪   → 测量角速度   → 积分 → 姿态角
```

### **关键特点**
- ✅ **自主性强**：不依赖外部信号（GPS等）
- ⚠️ **误差累积**：长时间运行会有误差漂移
- 🔄 **需要校正**：结合其他传感器（DVL、压力、磁力计）

---

## 🎯 本项目实现

### **传感器配置**

| 传感器 | 作用 | 融合方式 |
|--------|------|---------|
| **IMU** | 提供加速度和角速度 | 核心传感器，用于预测 |
| **DVL** | 提供速度测量 | 校正速度估计 |
| **压力传感器** | 提供深度 | 校正垂直位置 |
| **磁力计** | 提供航向角 | 校正偏航角 |
| **GPS** | 提供绝对位置 | 仅水面可用（水下不使用） |

### **算法：扩展卡尔曼滤波（EKF）**

```
预测步骤（Predict）:
├── 使用IMU加速度 → 更新位置和速度
├── 使用IMU角速度 → 更新姿态角
└── 更新协方差矩阵

更新步骤（Update）:
├── DVL速度测量 → 校正速度估计
├── 压力传感器深度 → 校正深度估计
├── 磁力计航向 → 校正航向角
└── GPS位置（水面）→ 校正位置估计
```

## 🔧 参数调优

### **调整传感器权重**

在 `simple_ins_node.py` 中：

```python
# DVL测量噪声（越小越信任）
R_vel = np.eye(3) * 0.01  # 默认0.01

# 压力传感器噪声
R_depth = np.eye(1) * 0.1  # 默认0.1

# 磁力计噪声
R_yaw = np.eye(1) * 0.05  # 默认0.05
```

**调优建议**：
- 降低噪声值 → 更信任该传感器
- 增加噪声值 → 减少该传感器影响

### **调整过程噪声**

```python
# 过程噪声（模型不确定性）
Q = np.eye(9) * 0.001  # 默认0.001
```

**调优建议**：
- 动态环境 → 增大过程噪声
- 静态环境 → 减小过程噪声

## 💡 常见问题

### **Q1: INS误差很大（> 1m）**

**可能原因**：
- 传感器未校准
- 过程噪声设置不当
- 某个传感器故障

**解决方法**：
```bash
# 1. 检查传感器数据
ros2 topic echo /rexrov/imu
ros2 topic echo /rexrov/dvl

# 2. 检查是否有故障注入
python3 interactive_fault_injector_fixed.py
请输入命令: 0  # 清除所有故障

# 3. 调整噪声参数
```

### **Q2: INS不工作（无输出）**

**可能原因**：
- INS节点未启动
- 传感器话题未发布

**解决方法**：
```bash
# 检查INS节点
ros2 node list | grep ins

# 检查传感器话题
ros2 topic list | grep rexrov

# 重启INS节点
python3 simple_ins_node.py
```

### **Q3: 位置误差随时间累积**

**正常现象！** 纯IMU导航必然会有误差累积。

**解决方法**：
- 增加DVL校正频率
- 增加压力传感器校正
- 定期使用GPS校正（水面）

## 📈 性能对比

| 导航方式 | 位置精度 | 速度精度 | 误差特性 | 适用场景 |
|---------|---------|---------|---------|---------|
| **纯IMU** | ~10 m/min | ~0.5 m/s | 快速累积 | 短时间导航 |
| **IMU + DVL** | ~1 m/min | ~0.1 m/s | 缓慢累积 | 中时间导航 |
| **融合INS** | ~0.1 m/min | ~0.05 m/s | 有界误差 | 长时间导航 |
| **真值** | 完美 | 完美 | 无误差 | 仿真测试 |

## 🎓 总结

### **✅ 优势**
- 自主导航，不依赖外部信号
- 融合多传感器，精度高
- 误差有界，适合长时间运行

### **⚠️ 限制**
- 需要多个传感器
- 参数调优复杂
- 计算量较大

### **🎯 适用场景**
- 水下机器人导航
- GPS拒止环境
- 故障注入测试
- 传感器融合研究
