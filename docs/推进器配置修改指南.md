## 推进器配置文件说明

### 文件位置
- **源文件**：`src/uuv_descriptions/urdf/rexrov_actuators.xacro`
- **安装文件**：`install/uuv_descriptions/share/uuv_descriptions/urdf/rexrov_actuators.xacro`

### 文件结构
```xml
<robot xmlns:xacro="http://www.ros.org/wiki/xacro">
  <!-- 推进器参数定义 -->
  <xacro:macro name="thruster_macro" params="namespace thruster_id *origin">
    <xacro:thruster_module_first_order_basic_fcn_macro
      namespace="${namespace}"
      thruster_id="${thruster_id}"
      mesh_filename="${prop_mesh_file}"
      dyn_time_constant="0.05"      <!-- 动态时间常数 -->
      rotor_constant="0.00031">     <!-- 转子常数（推力系数）-->
      <xacro:insert_block name="origin"/>
    </xacro:thruster_module_first_order_basic_fcn_macro>
  </xacro:macro>

  <!-- 推进器定义 -->
  <xacro:thruster_macro namespace="${namespace}" thruster_id="0">
    <origin xyz="X坐标 Y坐标 Z坐标" rpy="Roll Pitch Yaw"/>
  </xacro:thruster_macro>
  <!-- ... 更多推进器 ... -->
</robot>
```

### 参数说明
| 参数 | 含义 | 单位 | 范围 |
|------|------|------|------|
| `xyz` | 推进器位置（相对于机器人质心） | 米 | 任意实数 |
| `rpy` | 推进器方向（Roll翻滚, Pitch俯仰, Yaw偏航） | 度 | -180~180 |
| `dyn_time_constant` | 动态时间常数（响应速度） | 秒 | >0 |
| `rotor_constant` | 转子常数（推力系数） | N/(rad/s)² | >0 |

### 坐标系说明
- **X轴**：机器人前方为正
- **Y轴**：机器人左方为正
- **Z轴**：机器上方为正

## TAM矩阵详解

### 什么是TAM矩阵？

**TAM (Thruster Allocation Matrix)** = 推进器分配矩阵

**作用：** 将控制器的6自由度输出转换为各推进器的推力指令

```
控制器输出 (6×1)    TAM矩阵 (6×8)     推进器指令 (8×1)
[前后力  ]          ┌────────┐        [推进器0]
[左右力  ]    →     │ 6×8矩阵 │  →    [推进器1]
[上下力  ]          │        │        [推进器2]
[翻滚力矩]    ×     │        │        [推进器3]
[俯仰力矩]          └────────┘        [推进器4]
[偏航力矩]                            [推进器5]
                                     [推进器6]
                                     [推进器7]
```

### RexROV 8推进器TAM矩阵

```
        T0      T1      T2      T3      T4      T5      T6      T7
Fx  [ 0.160,  0.160, -0.160, -0.160,  0.707,  0.707, -0.707, -0.707]
Fy  [-0.214,  0.214, -0.214,  0.214,  0.707, -0.707,  0.707, -0.707]
Fz  [ 0.964,  0.964,  0.964,  0.964,  0.000,  0.000,  0.000,  0.000]
Mx  [ 0.435, -0.435,  0.435, -0.435, -0.091,  0.091, -0.091,  0.091]
My  [ 0.943,  0.943, -0.943, -0.943,  0.091,  0.091, -0.091, -0.091]
Mz  [ 0.137, -0.137, -0.137,  0.137, -0.649,  0.649,  0.649, -0.649]
```

**物理意义：**
- **Fx/Fy/Fz**：推进器在前后/左右/上下方向的分量
- **Mx/My/Mz**：推进器产生的翻滚/俯仰/偏航力矩

### 如何修改TAM矩阵

#### 删除推进器（8→4推进器）
```yaml
tam: [
  # 只保留前4列（T0-T3），删除后4列
  0.160, 0.160, -0.160, -0.160,           # surge
  -0.214, 0.214, -0.214, 0.214,           # sway
  0.964, 0.964, 0.964, 0.964,            # heave
  0.435, -0.435, 0.435, -0.435,           # roll
  0.943, 0.943, -0.943, -0.943,           # pitch
  0.137, -0.137, -0.137,  0.137            # yaw
]
```

#### 添加推进器
- 在TAM矩阵末尾添加新列
- 计算新推进器对6个自由度的贡献

## 常见修改场景

### 场景1：修改推进器位置

**目的：** 调整推进器在机器人上的位置

**步骤：**
1. 修改 `xyz="X Y Z"` 参数
2. 更新TAM矩阵（因为力臂改变了）
3. 测试

**示例：**
```xml
<!-- 原来：推进器0在右后方 -->
<xacro:thruster_macro namespace="${namespace}" thruster_id="0">
  <origin xyz="-0.89 0.33 0.53" rpy="0 -74.53 -53.21"/>
</xacro:thruster_macro>

<!-- 修改后：移到正后方，水平朝后 -->
<xacro:thruster_macro namespace="${namespace}" thruster_id="0">
  <origin xyz="-1.0 0.0 0.0" rpy="0 0 180"/>
</xacro:thruster_macro>
```

### 场景2：增加推力

**目的：** 提高推进器的推力输出

**步骤：**
1. 修改 `rotor_constant` 参数
2. **不需要**更新TAM矩阵
3. 测试

**示例：**
```xml
<!-- 原来 -->
<rotor_constant="0.00031">

<!-- 修改后：增加约60%推力 -->
<rotor_constant="0.00050">
```

### 场景3：减少推进器数量（8→4）

**目的：** 简化配置，只保留必要的推进器

**步骤：**
1. 注释掉不需要的推进器
2. 更新TAM矩阵（删除对应的列）
3. 测试

**示例：**
```xml
<!--
<xacro:thruster_macro namespace="${namespace}" thruster_id="4">
  <origin xyz="-0.412125 0.505415 0.129" rpy="0 0 45"/>
</xacro:thruster_macro>
<xacro:thruster_macro namespace="${namespace}" thruster_id="5">
  <origin xyz="-0.412125 -0.505415 0.129" rpy="0 0 -45"/>
</xacro:thruster_macro>
<xacro:thruster_macro namespace="${namespace}" thruster_id="6">
  <origin xyz="0.412125 0.505415 0.129" rpy="0 0 135"/>
</xacro:thruster_macro>
<xacro:thruster_macro namespace="${namespace}" thruster_id="7">
  <origin xyz="0.412125 -0.505415 0.129" rpy="0 0 -135"/>
</xacro:thruster_macro>
-->
```

### 场景4：添加垂直推进器

**目的：** 增强垂直移动能力

**步骤：**
1. 添加新的推进器定义（id=8）
2. 在TAM矩阵添加新列
3. 测试

**示例：**
```xml
<!-- 推进器8：底部垂直推进器 -->
<xacro:thruster_macro namespace="${namespace}" thruster_id="8">
  <origin xyz="0.0 0.0 -0.5" rpy="0 0 0"/>
</xacro:thruster_macro>
```

## 故障排除

### 问题1：启动后找不到包

**错误信息：** `Package 'uuv_descriptions' not found`

**解决方案：**
```bash
# 确保在工作空间目录
cd /home/wei1367/ros2_ws

# 正确的启动命令
source install/setup.zsh
ros2 launch uuv_descriptions upload_rexrov_default.launch.py ...
```

### 问题2：推进器数量不对

**检查方法：**
```bash
# 查看所有推进器话题
ros2 topic list | grep thrusters

# 应该看到：
# /rexrov/thrusters/0/input  (如果存在)
# /rexrov/thrusters/7/input  (如果存在)
```

### 问题3：机器人不动或动作异常

**可能原因：**
- TAM矩阵没有更新
- 推进器ID不连续
- 推进器方向设置错误

**解决方案：**
1. 检查TAM矩阵是否匹配推进器配置
2. 确保 `thruster_id` 连续（0,1,2,3,...）
3. 验证推进器方向（rpy参数）

### 问题4：Gazebo崩溃

**解决方案：**
```bash
# 完全关闭Gazebo
killall -9 gzserver gzclient

# 清理Gazebo缓存
rm -rf ~/.gazebo/

# 重新启动
```

## 备份与恢复

### 自动备份脚本

```bash
#!/bin/bash
# save_backup.sh

BACKUP_DIR="$HOME/ros2_ws_backup/thruster_config"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份URDF文件
cp src/uuv_descriptions/urdf/rexrov_actuators.xacro \
   $BACKUP_DIR/rexrov_actuators_$DATE.xacro

# 备份TAM矩阵
cp src/uuv_control/uuv_thruster_manager/config/rexrov/TAM.yaml \
   $BACKUP_DIR/TAM_$DATE.yaml

echo "备份完成: $BACKUP_DIR"
```

### 恢复备份

```bash
# 列出所有备份
ls -lh ~/ros2_ws_backup/thruster_config/

# 恢复特定备份
cp ~/ros2_ws_backup/thruster_config/rexrov_actuators_YYYYMMDD.xacro \
   src/uuv_descriptions/urdf/rexrov_actuators.xacro

# 更新到install目录
cp src/uuv_descriptions/urdf/rexrov_actuators.xacro \
   install/uuv_descriptions/share/uuv_descriptions/urdf/
```

| 修改类型 | 修改内容 | 更新TAM？ | 更新install？ |

## 总结

1. **备份优先**：修改前一定要备份原文件
2. **TAM很重要**：位置/方向改变后必须更新TAM矩阵
3. **测试验证**：修改后要在Gazebo中测试效果
4. **逐步进行**：一次只修改一个参数，便于排查问题
