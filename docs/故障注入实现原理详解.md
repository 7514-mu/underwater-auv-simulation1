## 故障注入可行性分析

### ✅ 可直接实现的故障

| 故障类型 | 可行性 | 原因 |
|---------|--------|------|
| 水下湍流 | ✅ | 通过`/apply_link_wrench`施加随机力 |
| 磁场干扰 | ✅ | 通过`/apply_link_wrench`施加周期力 |
| 压力舱进水 | ✅ | 通过`/apply_link_wrench`施加恒定向下力 |
| 传感器漂移 | ✅ | 通过`/apply_link_wrench`施加恒定向侧力 |

### ⚠️ 需要额外实现的故障

| 故障类型 | 可行性 | 问题 |
|---------|--------|------|
| 水流温度 | ⚠️ | 需要水动力插件支持运行时修改密度 |
| 通信延迟 | ⚠️ | 需要在推进器管理器层面拦截发布 |

### 📋 可行性总结

**可以直接使用：** 湍流、磁场、压力舱、传感器漂移（4种）
**需要改进：** 温度、通信延迟（2种）

## 各故障类型实现原理

### 1️⃣ 水下湍流干扰

#### 实现原理
通过Gazebo的`/apply_link_wrench`服务，向机器人施加随机变化的力和力矩

#### 核心代码逻辑
```python
# 步骤1：创建服务客户端
self.wrench_client = self.create_client(
    ApplyLinkWrench,
    '/apply_link_wrench'
)

# 步骤2：生成随机力
force_x = random.uniform(-magnitude, magnitude)  # 例如 -15 到 15 N
force_y = random.uniform(-magnitude, magnitude)
force_z = random.uniform(-magnitude/2, magnitude/2)

torque_x = random.uniform(-magnitude/2, magnitude/2)
torque_y = random.uniform(-magnitude/2, magnitude/2)
torque_z = random.uniform(-magnitude, magnitude)

# 步骤3：构造Wrench消息
wrench = Wrench(
    force=Vector3(x=force_x, y=force_y, z=force_z),
    torque=Vector3(x=torque_x, y=torque_y, z=torque_z)
)

# 步骤4：构造服务请求
request = ApplyLinkWrench.Request()
request.link_name = 'rexrov::base_link'
request.wrench = wrench
request.duration = Duration(sec=0, nanosec=500000000)  # 0.5秒

# 步骤5：调用服务
self.wrench_client.call_async(request)
```

#### 力的变化曲线
```
力(N)
 30 |      ┌──┐      ┌──┐
    │   ┌──┘  └──┐  ┌──┘  └──┐
 15 ├──┘        └──┘        └──┐
    │                            └──┐
  0 ├──┬────┬────┬────┬────┬────┬─→
    │  │    │    │    │    │    │
-15 │  └────┘    └────┘    └────┘
    │
-30 │
    └─────────────────────────────→ 时间(s)
      0    0.5  1.0  1.5  2.0  2.5
```

#### 为什么有效
- **真实物理**：Gazebo会根据F=ma计算加速度
- **叠加效果**：干扰力 + 推进器推力 = 实际受力
- **控制器响应**：PID检测到位置偏差，自动调整推进器

### 2️⃣ 磁场干扰

#### 实现原理
通过`/apply_link_wrench`施加周期性变化的力（正弦波）

#### 核心代码逻辑
```python
# 步骤1：设置参数
force_mag = 20.0      # 力幅值 20N
freq = 1.0            # 频率 1Hz
start_time = time.time()

# 步骤2：周期性计算力
while time.time() - start_time < duration:
    t = time.time() - start_time

    # 正弦波力（X和Y相位差90度，产生旋转效果）
    force_x = force_mag * math.sin(2 * math.pi * freq * t)
    force_y = force_mag * math.cos(2 * math.pi * freq * t)

    # 步骤3-5：同湍流干扰，构造请求并调用
    wrench = Wrench(
        force=Vector3(x=force_x, y=force_y, z=0.0),
        torque=Vector3(x=0.0, y=0.0, z=0.0)
    )
    # ... 调用服务 ...
    time.sleep(0.1)  # 每0.1秒更新
```

#### 力的变化曲线
```
力(N)
 20 |    ┌───┐        ┌───┐
    │  ╱     ╲      ╱     ╲
 10 |╱         ╲  ╱         ╲
    │           ╳            ╳
  0 |───────╲   ╳   ╲───────╳───→
    │         ╲ ╳     ╲     ╳
-10 │          ╳       ╲   ╳
    │        ╱ ╳         ╲ ╳
-20 |      ╱   └──────────╳─────→
    │
    └──────────────────────────→ 时间(s)
      0    0.25 0.5  0.75 1.0

force_x = sin波，force_y = cos波
```

#### 为什么有效
- **周期性振荡**：机器人会被周期性"拉来拉去"
- **共振风险**：如果频率接近机器人固有频率，振荡会放大
- **测试重点**：滤波器和振动抑制算法

### 3️⃣ 压力舱进水干扰

#### 实现原理
通过`/apply_link_wrench`施加恒定向下的力，模拟质量增加

#### 核心代码逻辑
```python
# 步骤1：计算进水质量
leak_depth = 0.5           # 进水深度0.5米
area = 0.5                 # 压力舱截面积0.5平方米
water_density = 1028.0     # 海水密度

added_mass = area * leak_depth * water_density  # 257kg

# 步骤2：计算增加的重力
added_gravity = added_mass * 9.81  # 2521N

# 步骤3：构造恒定向下力
wrench = Wrench(
    force=Vector3(x=0.0, y=0.0, z=-added_gravity),  # 负Z向下
    torque=Vector3(x=0.0, y=0.0, z=0.0)
)

# 步骤4：持续施加15秒
for _ in range(int(15.0 / 0.5)):  # 15秒，每0.5秒一次
    request = ApplyLinkWrench.Request()
    request.link_name = 'rexrov::base_link'
    request.wrench = wrench
    request.duration = Duration(sec=0, nanosec=500000000)

    self.wrench_client.call_async(request)
    time.sleep(0.5)
```

#### 力的变化曲线
```
力(N)
    0 │ ─────────────────────────────────
       │
-1000  │
       │
-2000  │
       │
-2521  │ ████████████████████████████████
       │
-3000  │
       └────────────────────────────────────→ 时间(s)
         0    2    4    6    8   10   12   15

恒定向下力（重力方向）
```

#### 为什么有效
- **F = ma**：向下的力让机器人加速下沉
- **控制器补偿**：PID检测到深度下降，增加垂直推进器推力
- **真实模拟**：等同于质量突然增加257kg

### 4️⃣ 传感器漂移干扰

#### 实现原理
通过`/apply_link_wrench`施加恒定侧向力，模拟传感器偏差

#### 核心代码逻辑
```python
# 步骤1：选择漂移方向和大小
drift_rate = "forward_0.1"  # 向前漂移0.1m/s
drift_force = 1.0            # 施加1N的力

# 步骤2：构造恒定向前的力
if "forward" in drift_rate:
    wrench = Wrench(
        force=Vector3(x=drift_force, y=0.0, z=0.0),  # X方向
        torque=Vector3(x=0.0, y=0.0, z=0.0)
    )
elif "lateral" in drift_rate:
    wrench = Wrench(
        force=Vector3(x=0.0, y=drift_force, z=0.0),  # Y方向
        torque=Vector3(x=0.0, y=0.0, z=0.0)
    )

# 步骤3：持续施加10秒
for _ in range(int(10.0 / 0.1)):
    request = ApplyLinkWrench.Request()
    request.link_name = 'rexrov::base_link'
    request.wrench = wrench
    request.duration = Duration(sec=0, nanosec=100000000)  # 0.1秒

    self.wrench_client.call_async(request)
    time.sleep(0.1)
```

#### 力的变化曲线
```
力(N)
 1.0 │ ████████████████████████████████
     │
 0.5 │
     │
 0.0 │ ─────────────────────────────────────→
     │
-0.5 │
     │
-1.0 │
     └────────────────────────────────────→ 时间(s)
       0    2    4    6    8   10

恒定向前的1N力
```

#### 为什么有效
- **恒定力**：机器人会持续向一个方向加速
- **传感器困惑**：GPS/深度计读数与预期不符
- **导航误差**：实际位置与控制算法认为的位置产生偏差

### 6️⃣ 通信延迟干扰（需改进）

#### 当前实现问题
```python
# 当前代码：只是在故障注入器中sleep，不影响实际控制
def inject_comm_delay_fault(self, packet_loss_rate):

        time.sleep(0.2)  # ❌ 只是让故障注入器sleep
    else:
```

#### 正确实现方案

**方案1：在故障注入器中订阅并转发（推荐）**
```python
class CommDelayFaultInjector(Node):
    def __init__(self, delay_rate):
        super().__init__('comm_delay_injector')

        # 订阅控制器发出的推进器指令
        self.input_subs = []
        for i in range(8):
            sub = self.create_subscription(
                Float64,
                f'/rexrov/thrusters/{i}/input',
                lambda msg, i=i: self.thruster_callback(i, msg),
                10
            )
            self.input_subs.append(sub)

        # 重新发布（延迟后）
        self.output_pubs = []
        for i in range(8):
            pub = self.create_publisher(
                Float64,
                f'/rexrov/thrusters/{i}/input_delayed',  # 新话题
                10
            )
            self.output_pubs.append(pub)

        self.delay_rate = delay_rate

    def thruster_callback(self, idx, msg):
        """拦截推进器指令，人为延迟后发布"""
        import random
        import time

        # 根据丢包率决定是否延迟
        if random.uniform(0, 100) < self.delay_rate:
            time.sleep(0.2)  # 延迟200ms
        else:
            time.sleep(0.01)  # 正常10ms

        # 发布到新话题
        self.output_pubs[idx].publish(msg)

# 使用时需要：
# 1. 控制器发布到 /input 原话题改为 /input_delayed
# 2. 或者使用remap
```

## 代码实现详解

### /apply_link_wrench 服务详解

#### 服务定义
```python
# gazebo_msgs/srv/ApplyLinkWrench
string link_name              # 必需：Gazebo链接名称，如 "rexrov::base_link"
string reference_frame        # 可选：参考坐标系，默认世界坐标系
geometry_msgs/Point reference_point  # 可选：力的作用点
geometry_msgs/Wrench wrench   # 必需：力和力矩
builtin_interfaces/Time start_time    # 可选：开始时间
builtin_interfaces/Duration duration  # 可选：持续时间，负数表示永久
---
bool success                  # 返回：是否成功
string status_message         # 返回：状态信息
```

#### 完整调用示例
```python
from gazebo_msgs.srv import ApplyLinkWrench
from geometry_msgs.msg import Wrench, Vector3
from builtin_interfaces.msg import Duration

# 创建请求
request = ApplyLinkWrench.Request()

# 必需参数
request.link_name = 'rexrov::base_link'  # 目标链接
request.wrench = Wrench(
    force=Vector3(x=10.0, y=0.0, z=0.0),   # 10N向前
    torque=Vector3(x=0.0, y=0.0, z=0.0)   # 无力矩
)

# 可选参数
request.reference_frame = ''  # 空字符串=世界坐标系
request.reference_point = Point(x=0.0, y=0.0, z=0.0)  # 质心
request.duration = Duration(sec=1, nanosec=0)  # 持续1秒

# 异步调用
future = self.wrench_client.call_async(request)

# 同步调用（会阻塞）
# future = self.wrench_client.call_async(request)
# rclpy.spin_until_future_complete(self.node, future)
# response = future.result()
```

## 实现限制与改进方案

### 限制1：服务调用频率

**问题：** 调用`/apply_link_wrench`需要时间，高频调用可能失败

**解决方案：**
```python
# 使用同步调用+错误处理
try:
    future = self.wrench_client.call_async(request)
    rclpy.spin_until_future_complete(self.node, future, timeout_sec=0.1)
except Exception as e:
    self.get_logger().warn(f'服务调用失败: {e}')
```

### 限制2：力的持续时间

**问题：** `duration`参数设置不正确，力可能只施加一瞬间

**解决方案：**
```python
# 持续施加力的两种方式
# 方式1：使用duration参数
request.duration = Duration(sec=10, nanosec=0)  # 一次性施加10秒

# 方式2：循环调用
for _ in range(100):  # 每0.1秒调用一次，持续10秒
    request.duration = Duration(sec=0, nanosec=100000000)
    self.wrench_client.call_async(request)
    time.sleep(0.1)
```

### 限制3：多干扰同时施加

**问题：** 同时施加多个干扰可能相互抵消或放大

**解决方案：**
```python
# 使用线程池独立控制每个干扰
from concurrent.futures import ThreadPoolExecutor

class MultiFaultInjector(Node):
    def __init__(self):
        self.executor = ThreadPoolExecutor(max_workers=4)

    def inject_multiple_faults(self):
        # 并发施加多个干扰
        self.executor.submit(self.inject_turbulence, 'medium')
        self.executor.submit(self.inject_magnetic, 'low')
        self.executor.submit(self.inject_pressure, 'shallow')
```
