# 故障组合使用指南

## 系统启动

```bash
# 终端1: 启动Gazebo
source ~/.zshrc
ros2 launch uuv_gazebo uuv_rexrov_gazebo.launch.py

# 终端2: 启动控制器
source ~/.zshrc
ros2 run uuv_control_cascaded_pid uuv_control_cascaded_pid

# 终端3: 启动键盘控制
source ~/.zshrc
ros2 run uuv_teleop uuv_teleop_key

# 终端4: 启动故障注入系统
source ~/.zshrc
python3 ~/ros2_ws/interactive_fault_injector_fixed.py
```

## 故障类型

### 推进器故障（选项1-7）
- **选项1**: 推进器0完全失效
- **选项2**: 推进器1完全失效
- **选项3**: 推进器0和1同时失效
- **选项4**: 推进器0效率降低至30%
- **选项5**: 推进器0效率降低至10%
- **选项6**: 推进器2完全失效
- **选项7**: 推进器6完全失效（垂直推进器）

### 环境故障（选项10-14）✨ **可以组合**
- **选项10**: 水下湍流干扰
  - low: 轻度湍流（±20N）
  - medium: 中度湍流（±50N）
  - high: 强湍流（±100N）

- **选项11**: 磁场干扰
  - low: 弱磁场（30N，0.5Hz）
  - medium: 中等磁场（60N，1.0Hz）
  - high: 强磁场（100N，2.0Hz）

- **选项12**: 压力舱进水
  - shallow: 浅层进水（+50kg）
  - medium: 中层进水（+250kg）
  - deep: 深层进水（+500kg）

- **选项13**: 传感器漂移
  - forward: 向前漂移
  - lateral: 侧向漂移

- **选项14**: 水温变化
  - temp_high: 水温升高（效率降至70%）
  - temp_low: 水温降低（效率降至90%）

### 其他选项
- **选项0**: 清除所有故障
- **选项8**: 洋流干扰（1.5m/s侧向流）
- **选项9**: 外力干扰（模拟碰撞）
- **选项15**: 查看当前活动故障
- **选项16**: 停止所有环境故障（保留推进器故障）
- **m**: 显示菜单
- **q**: 退出系统

## 故障组合示例

### 示例1: 恶劣海况（湍流 + 磁场）
```
输入: 10
选择湍流强度: medium
→ 机器人开始随机摇摆

输入: 11
选择磁场强度: high
→ 机器人开始周期性摆动，叠加在湍流上

输入: 15
→ 显示2个活动故障
```

```
输入: 1
→ 推进器0失效

输入: 10
选择湍流强度: high
→ 推进器0失效的同时，强湍流干扰
→ 机器人运动能力严重受损
```

### 示例3: 多重设备故障
```
输入: 12
选择进水深度: deep
→ 压力舱进水500kg，机器人下沉

输入: 13
选择漂移类型: lateral
→ 传感器漂移，机器人侧向偏移

输入: 14
选择温度变化: temp_high
→ 水温升高，所有推进器效率降低至70%

输入: 15
→ 显示3个活动故障
```

### 示例4: 极端情况演示
```
输入: 3
→ 推进器0和1同时失效

输入: 10
选择: high
→ 强湍流

输入: 11
选择: medium
→ 磁场干扰

输入: 12
选择: deep
→ 压力舱严重进水

输入: 15
→ 显示4个活动故障：推进器失效 + 湍流 + 磁场 + 进水
→ 机器人几乎失去控制能力！
```

## 观察故障效果

### 在Gazebo中观察
- 机器人会摇摆、漂移、下沉
- 推进器故障时，机器人无法正常移动
- 多故障叠加时，效果更加明显

### 监控推进器指令
```bash
# 另开终端，监控推进器0的指令
ros2 topic echo /rexrov/thrusters/id_0/input
```

### 监控位置
```bash
# 监控机器人位置
ros2 topic echo /rexrov/pose_gt
```

## 演示技巧

### 展示控制器鲁棒性
```
1. 先展示正常控制（按W键移动）
2. 注入单个故障（如湍流low）
3. 展示控制器如何补偿
4. 逐渐增加故障强度
5. 展示控制器何时失效
```

### 展示故障诊断
```
1. 注入未知故障
2. 观察推进器指令模式
3. 根据模式判断故障类型
4. 使用选项15确认
```

### 展示容错控制
```
1. 注入推进器故障（选项1）
2. 展示剩余推进器如何重新分配
3. 注入环境干扰
4. 展示系统在多重故障下的表现
```

## 注意事项

1. **环境故障会自动停止**（超时后）
2. **相同故障不能重复注入**（会提示已在运行）
3. **选项0清除所有故障**，包括推进器和环境
4. **选项16只停止环境故障**，保留推进器故障
5. **推进器故障会一直保持**，直到用选项0清除

## 演示流程建议

### 基础演示（5分钟）
```
1. 正常控制
2. 单个推进器故障（选项1）
3. 环境故障（选项10 medium）
4. 故障组合（选项1 + 选项10）
5. 恢复正常（选项0）
```

### 高级演示（10分钟）
```
1. 正常控制
2. 单故障演示（逐个测试选项10-14）
3. 故障组合演示（湍流+磁场）
4. 推进器+环境组合（选项1 + 选项10）
5. 极端情况（选项3 + 选项10high + 选项11medium）
6. 恢复演示（选项16，然后选项0）
```

### 对比演示
```
1. 测试无故障时的控制性能
2. 测试单故障时的控制性能
3. 测试多故障时的控制性能
4. 对比分析控制器在不同情况下的表现
```
