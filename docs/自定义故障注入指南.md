## 概述

基于官方的 `uuv_world_ros_plugins` 和 `gazebo_msgs` 包，可以创建多种自定义干扰类型来测试ROV/AUV的鲁棒性。

**核心原理：** 通过调用Gazebo官方服务接口来施加各种干扰

---

## 可用的干扰类型

### 🔥 官方已支持的干扰

| 干扰类型 | 服务接口 | 实现原理 |
|---------|---------|---------|
| **水流速度干扰** | `/hydrodynamics/set_current_velocity` | 设置洋流速度 |
| **外力干扰** | `/apply_link_wrench` | 施加力和力矩 |

### 🆕 新增自定义干扰

#### 1️⃣ 水流温度干扰
- **原理：** 通过流体密度变化模拟温度影响
- **效果：** 温度↑ → 密度↓ → 浮力↓ → 下沉力↑
- **选项：**
  - `cold`: 水温降低10°C (密度增加，浮力增大)
  - `warm`: 水温升高10°C (密度减小，浮力减小)

#### 2️⃣ 水下湍流干扰
- **原理：** 施加随机变化的力/力矩
- **效果：** 机器人摇摆、振动、控制困难
- **选项：**
  - `low`: 轻度湍流 (±5N)
  - `medium`: 中度湍流 (±15N)
  - `high`: 强湍流 (±30N)

#### 3️⃣ 磁场干扰
- **原理：** 施加周期性外力
- **效果：** 持续周期性振荡
- **选项：**
  - `low`: 弱磁场 (10N, 0.5Hz)
  - `medium`: 中等磁场 (20N, 1.0Hz)
  - `high`: 强磁场 (40N, 2.0Hz)

#### 4️⃣ 压力舱进水干扰
- **原理：** 向下施加力模拟重量增加
- **效果：** 机器人下沉、需要更大推力
- **选项：**
  - `shallow`: 浅层进水 (0.1m, ~51kg)
  - `medium`: 中层进水 (0.5m, ~257kg)
  - `deep`: 深层进水 (1.0m, ~514kg)

#### 5️⃣ 通信延迟干扰
- **原理：** 间歇性停止发布指令
- **效果：** 控制延迟、响应慢
- **选项：**
  - `10`: 10%丢包率
  - `30`: 30%丢包率
  - `50`: 50%丢包率

#### 6️⃣ 传感器漂移干扰
- **原理：** 施加恒定小力模拟读数偏差
- **效果：** 位置/速度读数持续偏离
- **选项：**
  - `forward_0.1`: 向前漂移0.1m/s
  - `lateral_0.1`: 侧向漂移0.1m/s

### 方法2：单独脚本注入

#### 水流温度干扰
```python
#!/usr/bin/env python3
import rclpy
from uuv_world_ros_plugins_msgs.srv import SetCurrentVelocityModel

def inject_temperature_fault(temp_change):
    """注入温度干扰"""
    node = rclpy.create_node('temp_fault_injector')
    client = node.create_client(
        SetCurrentVelocityModel,
        '/rexrov/hydrodynamics/set_current_velocity_model'
    )
    
    # 计算密度变化
    base_density = 1028.0  # 20°C海水
    density_change = temp_change * 0.3  # 每1°C约0.3 kg/m³
    new_density = base_density - density_change
    
    # 调用服务（需要实现具体的Srv类型）
    print(f"注入温度变化: {temp_change}°C, 密度: {new_density:.2f}")
```

## 干扰原理解释

### 温度干扰原理

```
水温 → 影响流体密度 → 影响浮力
  ↓         ↓            ↓
20°C  →  1028 kg/m³  →  1862.87kg物体浮力
30°C  →  1025 kg/m³  →  浮力减小（可能下沉）
10°C  →  1031 kg/m³  →  浮力增大（更容易上浮）
```

**实际物理效果：**
- 温度升高 → 机器人需要更多推力维持深度
- 温度降低 → 机器人更容易上浮
- 影响所有水动力系数

### 湍流干扰原理

```
湍流 → 随机力场 → 控制器震荡
  ↓         ↓           ↓
水流不稳 → 力的大小方向随机 → PID调节频繁
```

**实际效果：**
- 推进器指令波动
- 位置误差增大
- 能耗增加

### 磁场干扰原理

```
磁场 → 电磁力 → 周期性外力
  ↓      ↓         ↓
环境磁 → 周期变化 → 持续振荡
```

**实际效果：**
- 机器人周期性摆动
- 姿态控制困难
- 可能激发共振

## 扩展方法

### 1. 修改Gazebo插件参数

**修改流体密度：**
```xml
<!-- 在rexrov.gazebo.xacro中 -->
<gazebo>
  <plugin name="rexrov_uuv_plugin" filename="libuuv_underwater_object_ros_plugin.so">
    <fluid_density>1028.0</fluid_density>  <!-- 可修改 -->
  </plugin>
</gazebo>
```

### 2. 使用参数服务器动态修改

```python
import rclpy
from rclpy.parameter import Parameter

def set_fluid_density(new_density):
    """动态设置流体密度"""
    node.set_parameter(
        Parameter('fluid_density', value=new_density)
    )
```

### 3. 创建自定义Gazebo插件

**高级方法：** 编写C++插件

```cpp
// 自定义温度干扰插件
class TemperatureDisturbancePlugin : public gazebo::ModelPlugin {
public:
  void Load(gazebo::physics::ModelPtr _model, sdf::ElementPtr _sdf) {
    // 订算温度对流体密度的影响
    // 修改浮力和阻力系数
  }
};
```

## 注意事项

⚠️ **重要提醒：**
1. 所有干扰都是通过外力/力矩实现，不是物理模拟
2. 温度干扰通过密度变化近似，非真实热力学
3. 某些干扰可能组合使用产生复合效果
4. 建议在安全水深测试，避免失控

✅ **安全建议：**
- 先在浅水测试（z=-5m）
- 逐步增加干扰强度
- 始终监控机器人状态
- 准备紧急上浮方案
