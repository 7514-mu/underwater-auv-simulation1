# 测试场景示例 (Test Scenarios)

本文档提供具体的测试场景示例，供实验参考。

---

## 场景1: 单推进器完全失效测试

**目标**: 验证AUV对单推进器失效的容错能力

**步骤**:
1. 让AUV保持稳定悬停
2. 注入故障: `inject 1 0 10`
3. 观察AUV行为

**观察要点**:
- AUV是否能维持稳定
- 是否出现偏航
- 速度下降多少
- 补偿效果如何

**预期结果**:
- ✅ AUV能继续前进（速度略降）
- ✅ AUV自动调整其他推进器补偿
- ⚠️ 可能出现轻微偏航

---

## 场景2: 垂直推进器失效（关键测试）

**目标**: 测试垂直推进器失效的后果

**步骤**:
1. 让AUV保持在-20m深度
2. 注入故障: `inject 1 6 10`
3. 观察深度变化

**观察要点**:
- AUV是否无法维持深度
- 下沉速率是多少
- 是否需要紧急上浮

**预期结果**:
- ❌ AUV无法维持深度
- ⬇️ AUV缓慢下沉（浮力不足）
- ⚠️ 需要紧急上浮程序

---

## 场景3: 推力反向测试（危险！）

**目标**: 测试推力反向故障的危险性

**步骤**:
1. 让AUV保持稳定
2. 注入故障: `inject 4 6 10`
3. 尝试上浮（按X键）
4. 观察实际运动方向

**观察要点**:
- 按上浮是否反而下潜
- 传感器数据是否一致
- 故障检测时间

**⚠️ 警告**: 这是最危险的故障！按上浮键反而下潜

---

## 场景4: 推进器卡死测试

**目标**: 验证推进器卡死的后果

**步骤**:
1. 让AUV保持前进（按住W）
2. 注入故障: `inject 3 2 8`
3. 观察漂移情况

**观察要点**:
- 漂移方向
- 漂移速率
- 补偿效果

**预期结果**:
- AUV会出现持续的单向漂移

---

## 场景5: 推力振荡测试

**目标**: 验证推力波动对控制稳定性的影响

**步骤**:
1. 让AUV保持悬停
2. 注入故障: `inject 5 0,1,2,3,4,5 12`
3. 观察抖动情况

**观察要点**:
- 位置抖动幅度
- 速度波动范围
- 控制系统响应

**预期结果**:
- AUV出现明显抖动
- 位置方差增大

---

## 场景6: 渐进退化测试

**目标**: 验证推进器性能逐渐下降的影响

**步骤**:
1. 让AUV执行巡航任务
2. 注入故障: `inject 6 0,1,2,3,4,5 20`
3. 记录性能下降曲线

**观察要点**:
- 性能下降趋势
- 临界效率点
- 任务完成能力

**预期结果**:
- 速度逐渐下降
- 最终仅剩20%能力

---

## 场景7: 多推进器失效测试

**目标**: 找出AUV的临界失效点

**步骤**:
```bash
# 逐步增加失效推进器数量
inject 1 0,1 10      # 2个推进器失效
inject 1 0,1,2 10    # 3个推进器失效
inject 1 0,1,2,3 10  # 4个推进器失效
```

**观察要点**:
- 每个阶段的可控性
- 速度保持率
- 临界失效点

**记录表**:

```
1个      |        |          |          |
2个      |        |          |          |
3个      |        |          |          |
4个      |        |          |          |
```

## 场景8: 关键推进器测试

**目标**: 确定最关键的推进器

**步骤**:
```bash
# 依次测试每个推进器
inject 1 0 10  # 前左Surge
inject 1 1 10  # 前右Surge
inject 1 2 10  # 中左Sway
inject 1 3 10  # 中右Sway
inject 1 4 10  # 后左Surge
inject 1 5 10  # 后右Surge
inject 1 6 10  # 垂直Heave
```

**观察要点**:

- 速度下降百分比

## 场景9: 组合故障测试

**目标**: 测试不同类型故障组合的影响

**步骤**:
```bash
# 完全失效 + 部分失效
inject 1 0 10
inject 2 1 10

# 推力反向 + 振荡
inject 4 6 10
inject 5 7 10
```

**观察要点**:
- 组合故障的影响程度
- 系统是否完全失控
- 恢复控制的可能性

---

## 场景10: 边界条件测试

**目标**: 测试极端情况

**步骤**:
```bash
# 极短时间故障
inject 1 0 0.1

# 极长时间故障
inject 1 0 100

# 全部推进器失效
inject 1 0,1,2,3,4,5,6,7 10
```

**观察要点**:
- 系统恢复能力
- 是否需要重启
- 数据记录完整性
