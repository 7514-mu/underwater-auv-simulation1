### ✅ 第6步：准备编译环境（WSL2关键步骤）

**预计时间**: 5-10分钟

**⚠️ 重要说明**: WSL2内存限制严重，本地编译需要增加swap

---

#### A1. 检查当前内存状态

```bash
free -h
```

**预期输出**：
```
              total        used        free      shared  buff/cache   available
Mem:            15Gi       2.0Gi        12Gi       8.0Mi       796Mi        13Gi
Swap:           16Gi          0B        16Gi
```

---

#### A2. 增加swap空间（必须）

**为什么需要swap？**
- WSL2默认内存有限（16GB）
- 编译大型项目（UUV Simulator）需要大量内存
- 没有足够swap会导致编译失败：`Killed signal`

**创建32GB swap文件**：

```bash
# 1. 创建swap文件
sudo fallocate -l 32G /swapfile

# 2. 设置权限
sudo chmod 600 /swapfile

# 3. 格式化为swap
sudo mkswap /swapfile

# 4. 启用swap
sudo swapon /swapfile

# 5. 验证swap已启用
free -h
```

**预期输出**：
```
Swap:           48Gi          0B        48Gi  # 原来的16Gi + 新增的32Gi
```

**（可选）永久启用swap**：
```bash
# 添加到/etc/fstab，重启后自动启用
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

**状态**: ✅ 完成（48GB swap可用）

### ✅ 第7步：编译项目

**预计时间**: 1-2小时（WSL2单线程）

#### 编译流程

**本地编译**（标准流程）⭐

```bash
cd ~/ros2_ws

# 方案A1：最保守（推荐WSL2用户）
colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release --parallel-workers 1

# 方案A2：中等速度（如果swap足够大）
colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release --parallel-workers 2

# 方案A3：最快（原生Linux，32GB+物理内存）
colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release
```

**编译时间估算**：
- 单线程：2-3小时
- 双线程：1-2小时
- 多线程：30-60分钟（原生Linux）

1. 增加swap到64GB
2. 使用单线程编译
3. 只编译核心包：

- 核心包：uuv_descriptions, uuv_gazebo_worlds, uuv_thruster_manager

## 遇到的问题及解决方案

### 问题1: GPG密钥错误

**错误信息**:
```
W: GPG error: http://packages.ros.org/ros2/ubuntu jammy InRelease:
NO_PUBKEY F42ED6FBAB17C654
```

**解决方案**:
```bash
# 重新添加GPG密钥
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

# 更新软件包索引
sudo apt update
```

**状态**: ✅ 已解决

---

### 问题2: setup.zsh不兼容

**错误信息**:
```
${(%):-%N}: bad substitution
```

**原因**: ROS 2默认使用 `setup.bash`（bash shell），zsh存在兼容性问题

**解决方案**:
```bash
# 使用setup.bash而不是setup.zsh
echo "source /opt/ros/humble/setup.bash" >> ~/.zshrc
source ~/.zshrc
```

**状态**: ✅ 已解决

---

### 问题3: WSL2编译问题

**问题**: 本地编译容易失败，需要大量内存和时间

**解决方案**: 
- ✅ 增加swap空间到32GB
- ✅ 使用单线程编译
- ✅ 编译时间1-2小时

**状态**: ✅ 已解决

---

### 问题4: URDF文件缺失

**现象**: 启动机器人时找不到URDF文件

**解决方案**: 手动生成URDF

```bash
mkdir -p ~/ros2_ws/install/uuv_descriptions/share/uuv_descriptions/robots/generated/rexrov
```

### 问题5: spawn_entity服务找不到

**错误**: `Service /gazebo/spawn_entity unavailable`
