## 🚀 快速启动（5步）

### 步骤1：启动Gazebo仿真环境
**终端1：**
```bash
source ~/.zshrc
ros2 launch uuv_gazebo_worlds ocean_waves.launch
```
✅ 看到：Gazebo窗口打开，海洋场景加载

---

### 步骤2：启动RexROV模型
**终端2：**
```bash
source ~/.zshrc
ros2 launch uuv_descriptions upload_rexrov_default.launch.py gazebo_namespace:="''"
```
✅ 看到：RexROV AUV模型出现在Gazebo中

---

### 步骤3：启动PID控制系统
**终端3：**
```bash
source ~/.zshrc
bash ~/ros2_ws/start_pid_control_quiet.sh
```
✅ 看到：PID控制器已启动提示

---

### 步骤4：启动键盘控制
**终端4：**
```bash
source ~/.zshrc
RCUTILS_LOGGING_SEVERITY=ERROR ros2 run uuv_teleop uuv_keyboard_teleop
```
✅ 看到：键盘控制操作提示
- `W/S`: 前进/后退
- `A/D`: 左右平移
- `Q/E`: 上升/下降

---

### 步骤5：启动故障测试系统

**终端5 - 故障注入器：**
```bash
source ~/.zshrc
python3 ~/ros2_ws/interactive_fault_injector_fixed.py
```
✅ 看到：交互菜单

**终端6 - 监控器：**
```bash
source ~/.zshrc
python3 ~/ros2_ws/working_disturbance_monitor.py
```
✅ 看到：监控器启动提示

**终端7 - 验证器（可选）：**
```bash
source ~/.zshrc
python3 ~/ros2_ws/fault_verifier.py
```
✅ 看到：推进器状态显示

## 📖 详细操作流程

### 测试推进器故障

**1. 选择故障类型**
在终端5，输入数字并回车：
```
请输入命令 [0-9, q]: 1
```

**预期输出：**
```
⚠️  注入故障：推进器0效率降低至0%（完全失效）
✅ 推进器0故障注入成功（效率=0%）
💡 现在可以按W键观察推进器0故障的影响
```

**2. 验证故障生效**
在终端7，观察状态变化：
```
⚠️ 推进器0效率变化: 100% → 0%
============================================================
🔍 推进器状态检查
============================================================
  T0: ❌OFF | 效率:   0.0% ⚠️故障
  T1: ✅ON | 效率: 100.0% ✓正常
  T2: ✅ON | 效率: 100.0% ✓正常
  ...
============================================================
⚠️  检测到故障！推进器性能受损！
```

**3. 测试AUV响应**
在终端4，按住 `W` 键5-10秒

**终端6显示监控数据：**
```
⏰   10.5秒 | 📍 X=105.32 Y=  8.06 Z=  -0.36
   🎮 Vx=  0.50 Vy=  0.00 Vz=  0.00
   🔄 ωx=  0.00 ωy=  0.00 ωz=  0.00
   ➡️ 中等控制 (0.50)
```

**观察AUV行为：**
- 正常：AUV直线前进
- 故障：AUV向右偏转（因为T0失效）

**4. 清除故障**
在终端5，输入：
```
请输入命令 [0-9, q]: 0
```

**预期输出：**
```
🔧 正在清除所有故障...
✅ 所有故障已清除
💡 现在可以按W键测试正常控制
```

## ⚠️ 故障排查

### 问题1：监控器没有输出
**检查：**
```bash
# 确认键盘控制话题有数据
ros2 topic echo /rexrov/cmd_vel --once
```

**解决：** 在终端4按方向键，确保有控制输入

---

### 问题2：故障验证器显示全正常
**可能原因：** 服务调用失败

**检查：**
```bash
# 直接调用服务测试
ros2 service call /rexrov/thrusters/id_0/set_thrust_force_efficiency \
  uuv_gazebo_ros_plugins_msgs/srv/SetThrusterEfficiency "{efficiency: 0.0}"
```

**解决：** 确保Gazebo和RexROV已启动

---

### 问题3：AUV不响应键盘输入
**检查：**
```bash
# 确认PID控制器运行
ros2 node list | grep control
```

**预期输出：**
```
/acceleration_control
/velocity_control
```

**解决：** 重启终端3的PID控制

---

### 问题4：服务调用超时
**症状：**
```
waiting for service to become available...
```

**解决：**
1. 确认Gazebo正在运行
2. 检查服务是否存在：
   ```bash
   ros2 service list | grep thruster
   ```
3. 重启故障注入器脚本

## 📝 注意事项

1. **每次测试后清除故障**（输入0），避免故障叠加
2. **按Ctrl+C退出时会自动清除故障**
3. **日志文件会累积**，定期清理 `disturbance_logs/`
4. **终端7验证器可选**，主要用于调试
5. **如果系统卡顿**，减少日志记录频率或关闭验证器
