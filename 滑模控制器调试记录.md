## 问题1：滑模控制器启动失败 - RuntimeError

### 错误信息
```
RuntimeError: K coefficients: 6 coefficients needed
```

### 原因分析
Launch文件中的参数格式不正确。ROS 2 Python节点无法将逗号分隔的字符串解析为整数数组。

**错误的参数格式**：
```xml
<arg name="K" default="5,5,5,5,5,5"/>
<param name="K" value="[$(var K)]"/>
```

这会导致参数被传递为字符串 `"5,5,5,5,5,5"` 而不是整数数组 `[5,5,5,5,5,5]`。

### 解决方案
修改launch文件中的参数格式，使用数组格式：

**修改后的正确格式**：
```xml
<arg name="K" default="[5,5,5,5,5,5]"/>
<arg name="Kd" default="[4118.98,4118.98,4118.98,8000.0,8000.0,8000.0]"/>
<arg name="Ki" default="[0.06144,0.06144,0.06144,0.078,0.078,0.078]"/>
<arg name="slope" default="[0.182,0.182,0.182,3.348,3.348,3.348]"/>

<param name="K" value="$(var K)"/>
<param name="Kd" value="$(var Kd)"/>
<param name="Ki" value="$(var Ki)"/>
<param name="slope" value="$(var slope)"/>
```

### 修改的文件
- `/home/wei1367/ros2_ws/src/uuv_control/uuv_trajectory_control/launch/rov_nmb_sm_controller.launch`
- `/home/wei1367/ros2_ws/install/uuv_trajectory_control/share/uuv_trajectory_control/launch/rov_nmb_sm_controller.launch`

**注意**：修改src目录后需要 `colcon build`，但install目录的launch文件不会自动更新，需要手动修改或删除build/install目录后重新编译。

## 关键经验教训

1. **ROS 2参数格式**：数组类型的参数必须在launch文件中使用方括号格式 `"[1,2,3]"`，不能是 `"1,2,3"`

2. **编译缓存**：修改launch文件后，如果发现修改不生效，尝试删除build和install目录后重新编译

3. **服务调试**：
   - 先用 `ros2 service list` 检查服务是否存在
   - 再用 `ros2 service type <service_name>` 检查服务类型
   - 查看控制器终端日志确认服务是否被调用

4. **参数验证**：控制器启动时会打印所有参数，这是验证参数是否正确加载的好方法
